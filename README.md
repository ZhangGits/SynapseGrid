# SynapseGrid

可审计的事件溯源 AI Agent 系统。

## 设计目标

解决传统 LLM 应用**不可审计、不可回退、不可验证**的三大痛点：

1. **不可审计**：传统 LLM 应用的黑盒输出无法追溯决策依据。SynapseGrid 将每个分析结果拆解为结构化的分析要点（Finding），记录其来源、论据和置信度，形成可审计的分析链条。
2. **不可回退**：一旦生成报告，无法针对单个分析要点进行修正。SynapseGrid 支持对任意分析要点进行回退，并基于剩余有效信息重新整理报告，无需从头开始。
3. **不可验证**：无法证明报告内容未被篡改。SynapseGrid 基于事件溯源架构，将每次操作记录为不可变事件，通过 Merkle 树构建完整性证明，支持第三方验证。

### 核心设计原则

- **事件溯源（Event Sourcing）**：系统状态不是直接存储，而是通过回放事件日志重建。每个事件都有唯一哈希和语义摘要，确保不可篡改。
- **分析要点级粒度**：报告由多个独立的分析要点组成，每个要点包含观点、论据、置信度和来源，支持细粒度的审计和回退。
- **多 Agent 协作**：研究、验证、综合三个阶段由不同 Agent 负责，各自可配置不同的 LLM 模型，实现专业化分工。
- **双模式输出**：根据场景需求选择 Markdown 报告模式（简洁文本）或画布可视化模式（论证结构图）。

## 功能特性

### 三阶段 Agent 流水线
- **研究阶段（Research）**：将用户问题拆解为多个分析维度，生成带置信度的分析要点
- **验证阶段（Verification）**：对每个分析要点进行四维度可信度评估（证据强度、来源可靠性、推理合理性、数据一致性），并检测要点间关系
- **综合阶段（Synthesis）**：整合所有分析要点生成最终报告，画布模式下额外生成论证结构图

每阶段可独立配置 LLM 模型，支持模型降级策略（如验证阶段自动切换到轻量模型以降低成本）。

### 事件溯源审计
所有操作记录为不可变事件流，支持：
- **完整回放**：从任意版本快照开始回放事件，重建历史状态
- **Merkle 验证**：每个事件参与计算 Merkle 树根哈希，支持内容完整性验证
- **版本管理**：每次报告生成/回退产生新版本，支持查看历史版本列表
- **审计导出**：导出完整的事件日志、Merkle 证明和内容签名

### 分析要点回退
- **细粒度回退**：针对单个分析要点进行回退，不影响其他要点
- **自动重新整理**：回退后基于剩余有效要点自动重新整理报告
- **模式适配**：Markdown 模式下直接重新生成文本报告，画布模式下额外重新生成论证结构

### 双模式输出
- **Markdown 模式**：生成结构化文本报告，适合快速阅读和分享
- **画布模式**：生成可视化论证结构图，展示分析要点间的支持、张力、矛盾关系，支持交互式探索

### 成本追踪
- 每个 Agent 独立记录 Token 消耗和 API 费用
- 支持按 Agent / 模型维度分析成本构成
- 自动成本估算和预算提醒

### 场景化模板
内置通用、金融、法律等分析模板，根据场景自动调整分析维度和评估标准。

## API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/tasks` | 创建分析任务 |
| GET | `/api/v1/tasks/{id}/progress` | 获取任务进度 |
| GET | `/api/v1/tasks/{id}/result` | 获取任务结果 |
| POST | `/api/v1/tasks/{id}/rollback` | 回退分析要点 |
| POST | `/api/v1/tasks/{id}/feedback` | 画布反馈 |
| POST | `/api/v1/tasks/{id}/resynthesize` | 重新综合 |
| GET | `/api/v1/tasks/{id}/versions` | 获取版本列表 |
| POST | `/api/v1/verify` | 验证内容完整性 |
