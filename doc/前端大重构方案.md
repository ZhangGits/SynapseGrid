# 前端大重构方案：对话模式 + 可切换 Markdown 预览

## 一、需求理解

| 需求 | 说明 |
|------|------|
| 对话历史左栏 | 左侧显示从打开网页至今的全部对话，每条包含用户输入 + AI 回答摘要/链接 |
| 输入框左下 | 输入框和 Run 按钮移到左下角，类似聊天输入 |
| 模型选择顶部 | Provider / Model / API Key / Base URL 保留在顶部 header 区域 |
| Markdown 渲染 | 不直接显示在左边，改为可点击的链接框（card），点击后在右侧打开 |
| 右侧切换 | 右侧默认显示审计/验证/回退面板；点击 Markdown 链接后切换为全屏渲染，右上角有叉号关闭按钮，关闭后恢复审计面板 |

## 二、不改后端

后端 API 行为完全不变。前端通过 `useState` 在本组件内维护对话历史列表（浏览器内存），不做任何持久化。

## 五、新 TypeScript 类型

```typescript
interface ChatMessage {
  id: string;            // uuid
  timestamp: number;     // Date.now()
  prompt: string;        // 用户输入原文
  response: TaskResponse; // 后端返回的完整 JSON（含 session_id、markdown、audit、lineage、rollback）
}
```

## 六、交互流程

1. 用户在左下 `ChatInput` 输入 + 点击 Run
2. `App` 调用 `/api/v1/tasks`，拿到 `TaskResponse`
3. 构造 `ChatMessage { id, timestamp, prompt, response }`，插入 `chatHistory` state 末尾
4. `ChatHistory` 遍历 `chatHistory`，每条渲染为一张卡片：用户提示词摘要 + "View Report →" 链接
5. 用户点击 "View Report" → 设置 `activeMessage: ChatMessage | null`
6. 右侧从 AuditPanel 切换为 `MarkdownOverlay`，显示 `activeMessage.response.markdown_content`
7. 用户点击右上角 [×] → `activeMessage = null` → 右侧回到 AuditPanel（关联到最后一条消息的 session）
8. 回退按钮始终关联到最后一条消息的 session（`chatHistory[last].response.session_id`）

## 七、右侧面板逻辑

```typescript
function RightPanel({ chatHistory, activeMessage, onClose, onRollback, loading }) {
  if (activeMessage) {
    return <MarkdownOverlay content={activeMessage.response.markdown_content} onClose={onClose} />;
  }
  const last = chatHistory[chatHistory.length - 1];
  if (!last) return <div className="empty">Run a task to see audit data.</div>;
  return (
    <>
      <AuditPanel response={last.response} />
      <RollbackControls rollbackable={last.response.rollback_options.rollbackable_findings} loading={loading} onRollback={onRollback} />
    </>
  );
}
```

## 十、回退功能对齐

回退操作始终针对当前 `chatHistory` 的最后一条消息。回退成功后更新 `chatHistory` 中最后一条的 `response`。

```typescript
async function rollback(findingId: string) {
  const last = chatHistory[chatHistory.length - 1];
  const updated = await rollbackFinding(last.response.session_id, findingId);
  setChatHistory(prev => {
    const next = [...prev];
    next[next.length - 1] = { ...next[next.length - 1], response: updated };
    return next;
  });
}
```
