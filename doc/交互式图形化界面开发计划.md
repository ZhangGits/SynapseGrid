# SynapseGrid 交互式图形化界面开发计划

**日期**: 2026-06-14
**基准**: 《基于自然语言与图形化交互双向反馈的 AI Agent 系统设计方案》+ 当前代码实际状态

---

## 一、核心原则

1. **现有 Markdown 模式完全不受影响**——所有现有对话、审计面板、回退控件保持不变
2. **交互式界面是另一种可选的输出模式**——用户在输入框旁边通过一个开关选择 Markdown 或 Canvas
3. **Research → Verification 管道完全共享**——只有 Synthesis 的最终输出格式和前端渲染不同
4. **不引入 Strict Mode / Schema-as-Code 工具链**——MVP 阶段 Pydantic 手动定义 + `json_object` 已验证足够

---

## 二、用户体验流程

```
用户在 ChatInput 输入问题
         │
         ▼
    ┌─────────────┐
    │ 输出模式开关  │  ← 新: Markdown / 画布 二选一 toggle
    └─────────────┘
         │
         ├── "Markdown" (默认) ──→ 现有流程，不变
         │
         └── "画布" ──→ 新流程:
                         1. Research → Verification (不变)
                         2. Synthesis 输出 UI Schema JSON
                         3. 前端 CanvasViewer 渲染交互式画布
                         4. 用户在画布上拖拽/点击/输入
                         5. 反馈通过 /feedback 端点 → LLM → Partial Update
                         6. 画布增量更新
```

## 四、前后端数据流

### 初始画布生成

```
前端 POST /api/v1/tasks  { ...llm, output_mode: "canvas" }
  → Orchestrator.run_task()
    → Research + Verification (不变)
    → SynthesisAgent.run() 
        system_prompt = CANVAS_SYSTEM_PROMPT  (要求输出 UI Schema JSON)
        output = { components: [...], layout_type: "grid" }
    → 返回 TaskResponse { canvas_schema: {...}, markdown_content: "..." }
  → 前端: CanvasViewer 解析 canvas_schema，渲染 react-grid-layout
```

### 用户反馈循环

```
用户在画布上拖拽组件 + 点击 KPI + 输入文字 "把图表改成红色"
  → IntentionAggregator 防抖 300ms → 打包为 Intent Payload

前端 POST /api/v1/feedback  {
  session_id: "...",
  intent: { type: "COMPOUND_ACTION", base_version: 1, targets: [...], action_log: [...], user_text: "..." }
}
  → Orchestrator.process_feedback()
    → 调用 LLM (同一会话的 s_llm)
        system_prompt = FEEDBACK_SYSTEM_PROMPT
        user_prompt = 当前 UI Schema + 意图
        output = { action: "partial_update", target_version: 2, updates: [...] }
    → 返回 Partial Update
  → 前端: mergePatch(currentSchema, partialUpdate) → 画布增量更新
```

## 五、UI Schema 组件定义（MVP 3 种）

沿用设计文档 § 4.1 的定义，但由 Pydantic 手动建模而非工具链生成：

| 组件 | 可交互操作 | visual_config |
|------|-----------|---------------|
| `markdown` | 拖拽移动、点击 | 无，纯文本 |
| `kpi_card` | 拖拽移动、点击 | prefix/suffix/precision/color |
| `bar_chart` | 拖拽移动、点击数据点 | color/show_legend/orientation/stacked |
