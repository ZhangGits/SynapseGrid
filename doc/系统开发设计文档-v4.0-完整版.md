## 1. 系统概述

### 1.1 项目定位

**SynapseGrid** 是一个可审计的 AI Agent 分析系统，核心特性包括：

- **双模式输出**：支持 Markdown 报告模式和 Canvas 论证结构模式
- **事件溯源**：所有操作以不可变事件日志形式持久化到 SQLite
- **三阶段 Agent 流水线**：Research → Verification → Synthesis
- **可审计性**：Merkle 树保证报告完整性，支持回退和重新综合
- **人机交互**：前端支持论证结构可视化、节点详情弹窗、审计面板

### 1.2 核心术语

| 术语 | 定义 |
|------|------|
| **Finding** | 分析要点，包含 id、claim、evidence、confidence 等 |
| **Event** | 不可变的领域事件（TaskStarted, FindingReported, SynthesisGenerated 等） |
| **Session** | 一次完整的分析任务，对应一个 SQLite 数据库 |
| **Projection** | 从事件流投影出的内存状态视图 |
| **Canvas Schema** | 论证结构的图数据（nodes + edges），用于可视化 |
| **Provenance** | 报告中 `<finding>` 标签的溯源标记 |
| **Rollback** | 回退某个 finding，重新生成报告 |

#### 3.1.2 任务编排器 (`app/core/orchestrator.py`)

**职责**：协调 Research → Verification → Synthesis 三阶段流水线

**核心方法**：

```python
class Orchestrator:
    async def run_task(self, request: TaskRequest) -> TaskResponse:
        """双路径执行：
        - markdown 模式: Research → Verification → MarkdownGenerated
        - canvas 模式: Research → Verification → Synthesis → SynthesisGenerated
        """
    
    async def resynthesize(self, ctx) -> TaskResponse:
        """基于未回退的 findings 重新执行 Synthesis"""
    
    async def rollback_finding(self, ctx, finding_id: str, reason: str) -> TaskResponse:
        """回退一个 finding，触发重新综合"""
```

**双路径分支逻辑**：
```
用户输入 → TaskStarted 事件
    ↓
ResearchAgent.run() → (raw_markdown, findings[])
    ↓
for each finding: FindingReported 事件
    ↓
VerificationAgent.run(findings) → (assessments[], relations[])
    ↓
for each assessment: CredibilityAssessed 事件
for each relation: RelationDetected 事件
    ↓
if output_mode == "markdown":
    PostProcessor.enrich() → enriched_markdown
    MarkdownGenerated 事件
else:
    SynthesisAgent.run() → (logic_graph, enriched_markdown, node_map)
    SynthesisGenerated 事件
    ↓
_build_response() → TaskResponse
```

#### 3.1.3 状态机投影 (`app/core/state_machine.py`)

**职责**：将事件流投影为可查询的内存状态

**核心类**：
```python
class SessionStateProjection:
    state = {
        "task": None,                    # 任务信息
        "findings": {},                  # 分析要点字典
        "conflicts": {},                 # 冲突字典
        "credibility_assessments": {},   # 可信度评估
        "relations": [],                 # finding 间关系
        "rolled_back_findings": [],      # 已回退 finding ID 列表
        "costs": [],                     # 成本记录
        "synthesis": None,               # 综合结果
        "agent_traces": [],              # Agent 执行追踪
        "canvas_schema": None,           # 画布模式 schema
    }
```

**事件投影规则**：

| 事件类型 | 状态变更 |
|----------|----------|
| `TaskStarted` | `state["task"] = payload` |
| `FindingReported` | `state["findings"][id] = finding` |
| `FindingRolledBack` | `findings[id]["rolled_back"] = True` |
| `CredibilityAssessed` | `credibility_assessments[id] = assessment` |
| `RelationDetected` | `relations.append(relation)` |
| `SynthesisGenerated` | `state["synthesis"] = payload` |
| `MarkdownGenerated` | `state["synthesis"] = payload; canvas_schema = None` |

**关键不变性**：
1. 先持久化：事件先写入 SQLite 再投影到内存
2. 每个版本都有 Merkle 跟踪
3. `get_state_at(version)` 实现快照加速的部分重放
4. `rebuild(events)` 实现崩溃恢复

#### 3.1.4 会话管理器 (`app/core/session_manager.py`)

**职责**：会话生命周期管理（创建、查询、追加、关闭、分支、清理）

**核心设计**：
- **会话级隔离**：每个会话一个 SQLite 数据库文件
- **单写者模式**：每个会话一个 `asyncio.Queue`，唯一接触 SQLite 的协程
- **自动清理**：`CleanupScheduler` 调度过期会话删除

**会话上下文**：
```python
@dataclass
class SessionContext:
    session_id: str
    event_store: SessionEventStore      # SQLite 后端
    projection: SessionStateProjection  # 内存状态
    lineage: SessionLineageGraph        # 谱系图
    write_queue: asyncio.Queue          # 单写者队列
    writer_task: asyncio.Task           # 写者协程
    created_at: datetime
```

**事件追加流程**：
```
append_event(ctx, event)
    │
    ├── 1. 将事件入队 (event, future)
    │
    ├── 2. _writer_loop 从队列取出
    │      └── event_store.append(event) → 分配 version + hash
    │      └── future.set_result(saved_event)
    │
    ├── 3. 等待 future 完成（事件已持久化）
    │
    ├── 4. projection.apply(saved_event) → 更新内存状态
    │
    ├── 5. lineage.apply(saved_event) → 更新谱系图
    │
    ├── 6. lineage.persist_to_store() → 谱系图序列化回 SQLite
    │
    └── 7. 定期快照（每 50 个事件）
```

#### 3.1.5 事件存储 (`app/infrastructure/event_store.py`)

**职责**：SQLite 后端的事件持久化

**数据库设计**：

```sql
-- 事件表（不可变日志）
CREATE TABLE events (
    version         INTEGER PRIMARY KEY,  -- 自增版本号
    event_type      TEXT NOT NULL,         -- 事件类型
    actor           TEXT NOT NULL,         -- 执行者（JSON 序列化的 LlmConfig）
    payload         TEXT NOT NULL,         -- 事件载荷（JSON）
    metadata        TEXT NOT NULL,         -- 元数据（JSON）
    semantic_digest TEXT NOT NULL,         -- 语义摘要
    event_hash      TEXT NOT NULL,         -- SHA-256 事件哈希
    created_at      TEXT NOT NULL          -- ISO 8601 时间戳
);

-- 快照表（加速恢复）
CREATE TABLE snapshots (
    version     INTEGER PRIMARY KEY,
    state       TEXT NOT NULL,             -- JSON 序列化状态
    merkle_root TEXT NOT NULL,
    created_at  TEXT NOT NULL
);

-- 谱系图表
CREATE TABLE lineage (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    data        TEXT NOT NULL,
    updated_at  TEXT NOT NULL
);
```

**SQLite 优化**：
```sql
PRAGMA journal_mode=WAL;           -- 读写不阻塞
PRAGMA wal_autocheckpoint=2000;    -- 每 2000 页检查点
PRAGMA cache_size=-64000;          -- 64MB 页面缓存
PRAGMA synchronous=NORMAL;         -- WAL 模式下足够安全
PRAGMA busy_timeout=5000;          -- 等待锁释放 5 秒
```

    │      输出四维度评分：evidence_strength, source_reliability,

## 4. 数据模型

### 4.1 核心 Schema

#### TaskRequest
```python
class TaskRequest(BaseModel):
    user_id: str = "user001"
    prompt: str
    llm: LlmConfig = Field(default_factory=LlmConfig)
    llm_research: LlmConfig | None = None
    llm_verification: LlmConfig | None = None
    llm_synthesis: LlmConfig | None = None
    template: str = "general"
    output_mode: str = "markdown"  # "markdown" | "canvas"
    conversation_history: str | None = None
```

#### TaskResponse
```python
class TaskResponse(BaseModel):
    session_id: str
    markdown_content: str           # 原始 Markdown
    enriched_markdown: str          # 带 provenance 标记的增强 Markdown
    canvas_schema: LogicGraph | None # Canvas 模式论证结构
    audit_metadata: AuditMetadata
    lineage_data: LineageData
    rollback_options: RollbackOptions
```

#### LogicGraph (Canvas Schema)
```python
class LogicNode(BaseModel):
    id: str
    label: str
    type: str  # "question" | "conclusion" | "claim" | "evidence"
    finding_id: str | None = None
    credibility: float = 0.5
    rolled_back: bool = False

class LogicEdge(BaseModel):
    from_id: str
    to_id: str
    type: str  # "supports" | "perspective_difference" | "tension" | "genuine_contradiction"
    description: str = ""

class LogicGraph(BaseModel):
    nodes: list[LogicNode]
    edges: list[LogicEdge]
    node_map: dict[str, str]  # finding_id → node_id
```

## 5. 关键算法

### 5.1 事件哈希计算

```python
def compute_hash(self) -> str:
    """计算事件的 SHA-256 哈希"""
    data = json.dumps({
        "event_type": self.event_type,
        "actor": self.actor.model_dump() if isinstance(self.actor, BaseModel) else str(self.actor),
        "payload": self.payload,
        "metadata": self.metadata,
        "semantic_digest": self.semantic_digest,
    }, sort_keys=True, ensure_ascii=False)
    return hashlib.sha256(data.encode("utf-8")).hexdigest()
```

### 5.2 Merkle 树构建

```python
class MerkleTree:
    def append(self, event_hash: str):
        self.leaves.append(event_hash)
        self._rebuild()
    
    def _rebuild(self):
        """自底向上重建 Merkle 树"""
        current = self.leaves[:]
        while len(current) > 1:
            next_level = []
            for i in range(0, len(current), 2):
                left = current[i]
                right = current[i + 1] if i + 1 < len(current) else left
                next_level.append(hashlib.sha256((left + right).encode()).hexdigest())
            current = next_level
        self.root_hash = current[0] if current else ""
```

### 5.3 状态投影算法

```python
def get_state_at(target_version, event_store):
    """快照加速的历史状态查询"""
    # 1. 查找最近快照
    snapshot = event_store.get_latest_snapshot(max_version=target_version)
    if snapshot:
        state = deepcopy(snapshot.state)
        start_version = snapshot.version
    else:
        state = {}  # 空状态
        start_version = 0
    
    # 2. 重放快照之后的事件
    tail_events = event_store.get_events(start_version + 1, target_version)
    for event in tail_events:
        _project(event, state)
    
    return deepcopy(state)
```

### 5.4 回退算法

```python
async def rollback_finding(ctx, finding_id, reason):
    # 1. 标记回退
    state["findings"][finding_id]["rolled_back"] = True
    
    # 2. 获取剩余有效 findings
    active_findings = [f for f in all_findings if not f.get("rolled_back")]
    
    # 3. 重新组织 Markdown
    new_markdown = await research_agent.rebuild(
        original_markdown, finding_id, reason, active_findings, llm
    )
    
    # 4. 重新综合（canvas 模式）
    if output_mode == "canvas":
        new_result = await synthesis_agent.run(
            new_markdown, active_findings, assessments, relations, llm, template
        )
    
    # 5. 追加恢复事件
    append_event(ctx, SynthesisRestored / MarkdownRestored)
```

### 5.5 成本估算算法

```python
def _estimate_tokens_from_cost(cost_usd: float, model: str) -> int:
    """根据费用和模型反推 Token 数"""
    pricing = MODEL_PRICING.get(model, {"input": 0.001, "output": 0.002})
    avg_price = (pricing["input"] + pricing["output"]) / 2 / 1000
    return max(1, int(cost_usd / avg_price))
```
