# 审计特性前端改进方案

## 目标

将侧栏从"技术调试面板"改造为"用户可理解的审计视图"，不改后端任何代码。

## 范围

仅改前端 4 个文件，删 0，不碰后端。全部数据来自已有 JSON 响应。

---

## 改动明细

### 1. 侧栏顶部摘要：哈希面板 → 自然语言报告卡

**现状**：`Merkle root: sha256:9e4a80...` 和 `Content sig: hmac:...`

**改为**：
```
此回答基于 5 条分析要点，经过 1 轮验证，发现 1 处矛盾。
平均置信度 0.88 · 耗时 13.2s · 报告可审计

报告标识: 20260612-9e4a80a7
```

实现：从 `audit_metadata` 中提取 `event_count`（区分事件类型计数）、`findings` 数量、`conflicts` 数量、`duration_seconds`、`merkle_root` 前 8 位 + 日期拼接。

### 2. Findings 展示：纯文本 → 可展开卡片 + 引用状态

**现状**：每条 finding 显示 `claim` + `source` + 置信度百分比。

**改为**：
- 默认只显示 claims 摘要（前 80 字符），点击展开全文
- 每条增加颜色标记：根据 `rollbacked` / `validated` 状态显示 🟢 已确认 / 🔴 已回退
- 如有 `conflicts`，在涉及到的 finding 卡片上增加 ⚠️ 标记

### 3. 事件溯源时间线（新增）

**现状**：无。

**改为**：在 side panel 中根据现有 events 数据构建一条迷你时间轴：

```
已为您拆解为 5 条分析要点，经过验证，发现 1 处矛盾，综合后生成最终回答。
平均置信度 0.88 · 耗时 13.2s · 该回答可验证
```

实现：从 `audit_metadata.findings` 获取中间结论计数，从 `audit_metadata.conflicts` 获取矛盾数，从 `audit_metadata.duration_seconds` 获取耗时。

### 4. 对话卡片增强摘要

**现状**：`Analysis complete · 19 events · 1,436 tokens`

**改为**：
```
5 findings 验证完毕 · 1 条冲突 · 13.2s
置信度 0.88 · 1,436 tokens
```

### 5. Rollback 交互：即时回退 → 确认对话框

**现状**：点击即回退，无确认。

**改为**：点击回退时弹出确认框，展示该 finding 的 claim 摘要和影响信息，确认后才执行。
